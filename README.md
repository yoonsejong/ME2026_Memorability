# MediaEval 2026 Commercial Video Memorability Prediction (Team: CSAIL-TCNJ)

Predicts the memorability score of short branded video clips using multi-modal
feature fusion and cross-validated model selection.

This code does *NOT* work off-the-shelf! You must first obtain the [VIDEM dataset](https://link.springer.com/chapter/10.1007/978-3-032-02725-2_4) and features
provided by MediaEval Task Organizers. It requires some paperwork for dataset access.

Use `run_sweep.py` to run all 1,650 experiments and reproduce `sweep_results.csv`.

Five ensemble predictions are generated by `batch_ensemble_predict.py` as
`results/predictions/ensemble_YYYYMMDD_HHMMSS.csv` (one per run).

## Project overview

Each video is represented by features from four modalities — hand-crafted
visual, deep visual, audio, and emotion/semantic — extracted offline and stored
as `.npy` files.  Two fusion strategies are evaluated:

| Strategy | Description |
|---|---|
| **Concat** | Early fusion: feature sets are concatenated into one vector. |
| **Kernel** | Late fusion: one RBF kernel per feature set, summed (MKL). |

Models are evaluated with 5-fold cross-validation using Spearman rank
correlation as the metric.  The best configurations are then trained on the
full devset and used to predict the testset.

## Dependencies

**Prediction pipeline** (all scripts in the root directory):

```
pip install numpy pandas scipy scikit-learn joblib
```

**Feature extraction** (`feature_extractor/` only):

```
pip install librosa spacy nrclex nltk textblob numpy psutil
python -m spacy download en_core_web_sm
python -m textblob.download_corpora
```

ffmpeg must also be installed and on PATH (used by `audio_c.py`):
- Windows: https://ffmpeg.org/download.html (add `bin/` to PATH)
- Linux: `sudo apt install ffmpeg`
- macOS: `brew install ffmpeg`

Python 3.9+ is required. No GPU is needed.

## Data layout

| Split | Videos | Labels |
|---|---|---|
| devset | 339 | `devset_videolist_GT.csv` (columns: `id`, `memorability_score`) |
| testset | 85 | none (prediction target) |

Ground-truth labels are continuous scores in [0, 1].

Each feature set is stored as one `.npy` file per video under
`{split}/features/{feature_name}/{video_id}.npy` (e.g.
`devset/features/DenseNet121/CbfCUSPN7KI.npy`).  Feature arrays are either
1-D global descriptors or 2-D `(3, D)` three-frame descriptors;
`feature_loader` flattens all arrays to 1-D before building the feature matrix.

## Workflow

```
0. feature_extractor/get_all_features.py
         ↓ produces devset/features/<name>/*.npy
           and testset/features/<name>/*.npy
         ↓
1. run_sweep.py          →  results/sweep_results.csv
         ↓
2. ensemble_cv.py        →  console report (validate ensemble by CV)
         ↓
3. ensemble_predict.py   →  results/predictions/ensemble_<ts>.csv (single config)
   batch_ensemble_predict.py  →  one CSV per run (multiple configs)
```

Step 0 is only needed if you are re-extracting TCNJ audio/emotion/semantic
features from raw video.  Steps 1 and 2 are for exploration / validation.
Step 3 produces submission-ready files.

## Entry points

### `run_sweep.py` — grid search

Evaluates every `(combination, selector, model)` triple with 5-fold CV and
saves Spearman scores to `results/sweep_results.csv`.  Supports resuming an
interrupted run.

```bash
# Full sweep (both fusion strategies, all combos/models/selectors)
python run_sweep.py

# Subset
python run_sweep.py --fusion kernel --models SVR Ridge --combos All ViT
python run_sweep.py --selectors none spearman_50
python run_sweep.py --jobs 4   # limit parallelism
```

**Options**

| Flag | Default | Values |
|---|---|---|
| `--combos` | all 33 | any combination name |
| `--models` | all 5 | `LinearRegression Ridge MLP SVR GradientBoosting` |
| `--selectors` | all 5 | `none spearman_30 spearman_50 pca_30 pca_50` |
| `--fusion` | `both` | `concat` `kernel` `both` |
| `--jobs / -j` | `-1` (all cores) | integer |

---

### `ensemble_cv.py` — ensemble validation

Runs 5-fold CV on a user-defined ensemble and prints per-model and fused
Spearman scores.  Edit `CONFIGURATIONS` at the top.

```bash
python ensemble_cv.py
```

---

### `ensemble_predict.py` — ensemble testset prediction

Trains the ensemble on the full devset and saves a timestamped CSV.
Edit `CONFIGURATIONS` at the top.

```bash
python ensemble_predict.py
```

Output: `results/predictions/ensemble_YYYYMMDD_HHMMSS.csv`
Output columns: `video, mem_score`

---

### `batch_ensemble_predict.py` — multiple ensembles

Edit `CONFIGURATIONS_LIST` at the top (each entry is an independent ensemble),
then run:

```bash
python batch_ensemble_predict.py
```

Produces one timestamped CSV per ensemble.

## Module reference

| Module | Responsibility |
|---|---|
| `feature_loader.py` | Load `.npy` files, flatten, inner-join, return aligned matrices |
| `feature_selector.py` | `CV_SELECTORS` (fold-safe) and `FULL_SELECTORS` (full-devset) registries |
| `models.py` | `get_pipeline(name)` — `StandardScaler + estimator` pipelines |
| `evaluate.py` | `evaluate_combo()` — concat-mode 5-fold CV |
| `kernel_fusion.py` | `evaluate_combo_kernel()` — kernel-mode 5-fold CV; `train_and_predict_kernel()` |

**Selectors**

| Key | Method |
|---|---|
| `none` | No selection |
| `spearman_30` / `spearman_50` | Top-30 / top-50 features by |Spearman r| with labels |
| `pca_30` / `pca_50` | PCA to 30 / 50 components |

All selectors are fitted on the training fold only.

**Fusion modes inside ensemble files**

`preprocessing: "concat"` — features concatenated, then model trains on the
flat vector.

`preprocessing: "kernel"` — one RBF kernel per feature set (selection and
scaling applied per-set), kernels summed, model trains on the kernel matrix.

## Feature extractor

`feature_extractor/` contains the extraction code for four modalities 
derived from raw video files and speech-to-text transcripts.

| Script | Output shape | Description |
|---|---|---|
| `audio_c.py` | (3, 9) | Temporal audio features (start/mid/end segments): pitch, energy, pause statistics |
| `audio_o.py` | (9,) | Global audio features over the full clip |
| `emotion.py` | (19,) | NRC emotion scores + language complexity from transcript |
| `semantic.py` | (7,) | Semantic category scores from transcript vocabulary |

**Inputs required** (not included in this repo):
- Raw video files (`.mp4`, `.mkv`, or `.webm`) in `video-devset/` and `video-testset/`
- Speech-to-text transcripts in `devset/devset-stt/` and `testset/testset-stt/`

**Run**:

```bash
cd feature_extractor
python get_all_features.py
```

This extracts features for both devset and testset, validates outputs, and
zips results to `feature_extractor/features.zip`.  The extracted `.npy` files
must then be placed in `devset/features/<feature_name>/` and
`testset/features/<feature_name>/` before running the prediction pipeline.

## Results

You can find all 5-fold CV results in `results/sweep_results.csv`.

Full sweep: 1,650 configurations (33 combinations × 5 selectors × 5 models × 2 fusion modes).

All ensemble predictions use rank-average fusion of per-model outputs.
